# easyp 0.1.2 Release
[draft release document]
We're excited to announce the release of easyp 0.1.2, now with fully automatic Acme certificates.

to give it a go 

```bash
curl www.easyp.net/easyp-0.1.2-x64.gz -o easyp.gz && gunzip easyp.gz && [ "$(sha256sum < easyp)" = "aea5971bda8959c73701ece1a2b14024a67ebb48c41bc5717f3274297fb6f07a  -" ] && chmod +x easyp && sudo ./easyp
```

## 📊 Performance Benchmarks

We conducted comprehensive performance testing comparing easyp against industry-standard web servers:

### Memory Usage Comparison

| Server | Memory Usage | Architecture |
|--------|-------------|--------------|
| **easyp 0.1.2** | **8.1 MB RSS** | Single process |
| Apache 2.4.58 | 36 MB RSS | Multi-process (master + workers) |

**Result**: easyp uses **78% less memory** than Apache.

### Throughput Performance

| Server | Requests/sec | Concurrency | Use Case |
|--------|-------------|-------------|----------|
| Apache 2.4.58 | 6,042 req/sec | 20 concurrent | High-traffic production |
| **easyp 0.1.2** | **77 req/sec** | **10 concurrent** | **Lightweight, embedded** |

Clearly we will have to debug why easyp is so slow. This is just a prototype release!


```bash
curl www.easyp.net/easyp-0.1.2.tgz -o easyp.tgz
tar -zxvf easp.tgz
cd easyp/easyp-crate
cargo build --release --bin easyp
```

## 🔮 What's Next

Looking ahead to future releases, we're planning:

- **HTTP/2 support**: Full HTTP/2 implementation with multiplexing
- Find out why it is so slow.
- Security Audit
- Supply security updates via some secure channel.
- Investigate feasibility of automatic free subdomain instead of self-signed cert fallback.
- Fix warnings (Unused code: use or remove)
- Add more configuration options (e.g. aws-lc-ls without RSA)
- Sync with upstream.
- Use Async/Await properly, don't fallback on polling.
- Check that configuration options really work. I have mostly just tested auto-ACME mode without special configuration.
- Detect file encoding (UTF8 vs UTF16 etc.) support unknown file types?
